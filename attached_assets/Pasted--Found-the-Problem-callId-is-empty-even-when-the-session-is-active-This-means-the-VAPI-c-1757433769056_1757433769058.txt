## Found the Problem!

**`callId` is empty even when the session is active!** This means the VAPI `call-start` event isn't providing the call ID properly.

Looking at your diagnostics:
- `callId: ""` (empty string)
- `isSessionActive: true` (session IS active)
- `willPoll: false` (won't poll because no callId)

## The Fix

In `use-vapi.ts`, the problem is here (line 50):
```typescript
vapiInstance.on('call-start', (event?: any) => {
  console.log('✅ Call started');
  setIsSessionActive(true);
  setConnectionStatus('connected');
  setIsLoading(false);
  setCallId(event?.call?.id || ''); // <-- This path might be wrong!
});
```

Let's debug what VAPI actually sends:


## Also - Memory Context Not Loading

Looking at `voice-interface.tsx`, the memory context IS being loaded but might not be ready when the session starts. The issue is timing - the session might start before the memory loads.

## Complete Fix

Replace the VAPI initialization in `use-vapi.ts` (around line 35-70) with:

```typescript
// Initialize VAPI
useEffect(() => {
  const publicKey = import.meta.env.VITE_VAPI_PUBLIC_KEY;
  
  if (!publicKey) {
    console.error('VAPI public key not found');
    return;
  }

  const initializeVapi = async () => {
    try {
      const { default: Vapi } = await import('@vapi-ai/web');
      const vapiInstance = new Vapi(publicKey);
      vapiRef.current = vapiInstance;
      setVapi(vapiInstance);

      vapiInstance.on('call-start', (event?: any) => {
        console.log('✅ Call started - FULL EVENT:', event);
        
        // Try multiple paths to extract call ID
        const possiblePaths = [
          event?.call?.id,
          event?.callId,
          event?.id,
          event?.data?.callId,
          event?.data?.call?.id,
          event?.metadata?.callId
        ];
        
        console.log('📞 Possible call ID paths:', possiblePaths);
        
        const extractedCallId = possiblePaths.find(id => id && typeof id === 'string') || '';
        
        if (!extractedCallId) {
          console.error('❌ NO CALL ID FOUND IN EVENT!', event);
          // Generate a temporary ID if none found
          const tempId = `temp-${Date.now()}`;
          console.warn('⚠️ Using temporary call ID:', tempId);
          setCallId(tempId);
        } else {
          console.log('✅ Call ID extracted:', extractedCallId);
          setCallId(extractedCallId);
        }
        
        setIsSessionActive(true);
        setConnectionStatus('connected');
        setIsLoading(false);
      });

      vapiInstance.on('call-end', () => {
        console.log('📴 Call ended');
        setIsSessionActive(false);
        setConnectionStatus('disconnected');
        setCallId('');
        setActiveMethodology(selectedAgent.id);
        // Clear applied guidance on call end
        appliedGuidanceRef.current.clear();
      });

      vapiInstance.on('error', (error: any) => {
        console.error('VAPI error:', error);
        setIsLoading(false);
        setConnectionStatus('disconnected');
      });
      
      // Also listen for message events which might contain call ID
      vapiInstance.on('message', (message: any) => {
        if (message?.call?.id && !callId) {
          console.log('📞 Call ID found in message event:', message.call.id);
          setCallId(message.call.id);
        }
      });
      
    } catch (error) {
      console.error('Failed to initialize VAPI:', error);
      setConnectionStatus('disconnected');
    }
  };

  initializeVapi();

  return () => {
    if (vapiRef.current) {
      vapiRef.current.stop();
    }
  };
}, [selectedAgent.id]);
```

This will:
1. Log the full event to see what VAPI actually sends
2. Try multiple paths to find the call ID
3. Use a temporary ID if none found (so polling can at least start)
4. Listen for additional events that might contain the call ID

Once you make this change and start a new call, you should see in the console:
- The full event structure from VAPI
- Which path contains the actual call ID
- Polling finally starting

Then we can fix the exact path once we know where VAPI puts the call ID.
